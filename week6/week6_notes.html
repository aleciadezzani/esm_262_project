Thinking Big: how to create modules as parts of a big program

A Simple Function Example - Just One Module

Let’s say we want to develop a program that computes power generated from a hydro-reservoir

We know that power generation depends on water release rates and the potential energy (from the reservoir height)

Let’s Think About a More Complex Program…

Goal = Calculate profit from a reservoir fed by a river, if a minimum reservoir volume must be maintained and otherwise the reservoir is operated at a flow rate set by the user

Input = Stream gage data (m^3^/sec)

Parameters = Efficiency, ρ (density of water), g (acceleration due to gravity), Maximum Reservoir Capacity (m3), Minimum Reservoir Volume (m3), Set flow rate (m3/sec), Price of power (/W)$,

Output = Profit ($)

Modules

  • Compute Power from a given reservoir height and flow rate (compute_power)
  • Keep track of reservoir volume given inflow and output (reservoir_state)
  • Determine flow rate, given operator desired flow rate but need for minimum capacity (determine_flow_rate)
  • Compute profit from power (compute_profit) Each module would have its own inputs, parameters and output

A Flow control diagram puts the modules together

Two ways in R

ifelse function - ifelse(condition, TRUE, FALSE) - best used for simple application

if..else structure more formal way of deciding on two paths
